# Apresentacao para LinkedIn

## Titulo sugerido

> Desenvolvi uma plataforma de farmacovigilancia para idosos baseada nos
> criterios oficiais AGS Beers 2023 — arquitetura completa, do banco de
> dados ao dashboard clinico.

(Alternativas: "Como a ciencia de dados pode reduzir reacoes adversas a
medicamentos em idosos" / "SeniorRx Monitor: de dado sintetico a alerta
clinico explicavel".)

## Corpo do post (modelo)

> Pacientes idosos com polifarmacia tem risco significativamente maior de
> reacoes adversas a medicamentos — e boa parte desse risco e conhecido e
> catalogado, mas raramente verificado de forma sistematica no momento da
> prescricao.
>
> Construi o **SeniorRx Monitor**, um projeto de portfolio que implementa os
> **AGS Beers Criteria(R) 2023** como um motor de regras auditavel, integrado
> a uma API REST e um dashboard clinico, com:
>
> - Deteccao de polifarmacia/hiperpolifarmacia, PIM, interacoes
>   medicamento-medicamento e ajustes por funcao renal — cada alerta
>   rastreavel a uma fonte cientifica especifica;
> - Arquitetura limpa (domain/application/infrastructure/interface) com
>   >80% de cobertura de testes;
> - Pipeline de MLOps completo: MLflow para tracking de modelo, Evidently AI
>   para deteccao de drift, CI/CD no GitHub Actions, Docker Compose;
> - Relatorio epidemiologico reprodutivel em R/Quarto.
>
> O objetivo nao e substituir julgamento clinico, mas mostrar como
> engenharia de software rigorosa + ciencia de dados podem apoiar decisoes
> de farmacoterapia com transparencia total sobre o "porque" de cada alerta.
>
> Codigo aberto (MIT), 100% dados sinteticos, documentacao tecnica completa.

o projeto e uma ferramenta de apoio a decisao nao validada clinicamente.
Prefira linguagem precisa: "identifica", "sinaliza", "apoia a revisao
farmacoterapeutica". Isso demonstra maturidade tecnica e etica, o que pesa
positivamente em avaliacoes tecnicas de HealthTech.
